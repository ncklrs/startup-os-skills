## Visual Storytelling - Show Don't Tell

**Impact: MEDIUM-HIGH**

Video is a visual medium. When you can show something, don't tell it. Great VSL scripts use visuals to communicate half the message, with copy reinforcing what viewers see.

## The Show/Tell Balance

### The 60/40 Rule

In product videos, aim for:
- **60% shown visually** (demo, transformation, before/after)
- **40% told with copy** (context, benefits, CTA)

### Good Example

```
Scene: Speed Feature (10s)

SHOW (60%):
  Visual: Split screen
    Left: Progress bar slowly filling (old way)
    Right: Instant completion (new way)
  Clock showing time: 45 minutes vs 2 minutes

TELL (40%):
  On-screen: "23x faster processing"

Why it works:
  Visual demonstrates speed difference
  Copy quantifies what viewers just saw
  Both reinforce same message
```

### Bad Example

```
Scene: Speed Feature (10s)

SHOW:
  Generic product screenshot (static)

TELL:
  On-screen: "Our product processes data 23x faster than
  traditional solutions using advanced algorithms and
  optimized architecture"

Why it fails:
  90% telling, 10% showing
  Viewers don't see speed, just read claim
  No visual proof
```

## Visual Story Techniques

### 1. Before/After Transformation

**Most powerful visual storytelling technique for product videos**

**Good Example**:
```
Scene: Transformation (12s)

Visual Sequence:
  Before (0-4s):
    - Cluttered, chaotic workspace
    - Multiple tabs, apps, sticky notes
    - Dark, messy colors (#666, #999)
    - Person looking stressed

  Transition (4-6s):
    - Smooth morph animation
    - Chaos organizing itself

  After (6-12s):
    - Clean, single interface
    - Organized, clear workflow
    - Bright, clean colors (#fff, #4A90E2)
    - Person relaxed, smiling

On-Screen: "One workspace. Everything organized."

Why it works:
  Story told primarily through visual contrast
  Copy reinforces but doesn't need to explain
  Emotional journey clear without words
```

### 2. Problem Visualization

**Show the pain point, don't just describe it**

**Good Example**:
```
Scene: Problem (8s)

Visual:
  - Designer switching between 12 different apps
  - Fast cuts showing app switching
  - Each switch shows loading spinner
  - Clock ticking, time wasting
  - Designer's frustration building

On-Screen: "47 tools. One project."

Why it works:
  Viewer feels the frustration visually
  Copy is minimal but memorable
  Exaggeration (47 tools) is visual, not just claim
```

**Bad Example**:
```
Scene: Problem (8s)

Visual:
  - Static image of person at desk

On-Screen:
  "Modern designers use too many disconnected tools which
  creates inefficiency and frustration in their workflow"

Why it fails:
  Entirely told, not shown
  Static visual adds nothing
  Paragraph of text does all the work
```

### 3. Demonstration in Action

**Show the product working, not screenshots**

**Good Example**:
```
Scene: Demo (15s)

Visual:
  - Mouse cursor enters frame
  - Types command: "/brand"
  - Instant dropdown with brand assets
  - Selects logo
  - Logo appears in design
  - All in real-time, no cuts

On-Screen: "Instant access to brand assets"

Why it works:
  Actual interaction shown
  Speed is visible, not claimed
  Viewer thinks "I could do that"
```

**Bad Example**:
```
Scene: Demo (15s)

Visual:
  - Static screenshot of interface
  - Red arrows pointing to features

On-Screen:
  "Click here to access brand assets which are available
  in the dropdown menu with fast loading times"

Why it fails:
  Screenshot doesn't demonstrate anything
  Have to imagine the interaction
  Text does all the explaining
```

### 4. Data Visualization

**Make numbers visual, not just text**

**Good Example**:
```
Scene: ROI Proof (10s)

Visual:
  - Bar graph animating
  - Before: Small bar (2 deals/month)
  - After: Large bar (7 deals/month)
  - Bars are actual product screenshots (deals in CRM)
  - Difference highlighted with motion

On-Screen: "3.5x more deals closed"

Why it works:
  Number becomes visual comparison
  Motion emphasizes growth
  Real product context (CRM deals)
```

**Bad Example**:
```
Scene: ROI Proof (10s)

Visual:
  - Plain background

On-Screen:
  "Teams using our product close 3.5x more deals per month
  compared to teams not using our product"

Why it fails:
  Pure text, no visual
  Claim without proof
  Missed opportunity to show growth
```

### 5. Metaphor and Analogy

**Use visual metaphors for abstract concepts**

**Good Example**:
```
Scene: Integration Feature (8s)

Visual:
  - Puzzle pieces scattered (individual tools)
  - Pieces snap together
  - Form complete picture
  - Picture reveals unified dashboard

On-Screen: "All your tools. One platform."

Why it works:
  Abstract concept (integration) made concrete
  Visual metaphor is clear
  Satisfying moment (puzzle completion)
```

**Use sparingly**: Metaphors can feel cheesy if overused

### 6. User Journey Visualization

**Show the path, not just the destination**

**Good Example**:
```
Scene: User Journey (20s)

Visual:
  Step 1 (0-6s): User signs up (form filling)
  Step 2 (6-12s): Connects first tool (OAuth flow)
  Step 3 (12-18s): First insight appears (data visualization)
  Step 4 (18-20s): User celebrates (visual success moment)

On-Screen:
  "From signup to insight in 5 minutes"

Why it works:
  Journey is shown, not described
  Each step is visual
  Time progression is clear
```

## Visual Hierarchy

### What to Show First

In each scene, establish visual hierarchy:

```
Primary (First 1s):
  What grabs attention immediately
  Usually: Key visual element or motion

Secondary (Next 1-2s):
  Supporting elements that add context

Tertiary (Final 1-2s):
  Details, text, fine print
```

**Good Example**:
```
Scene: Feature Reveal (5s)

Visual Hierarchy:
  Primary (0-1s):
    Large animation of data flowing into dashboard
    [Attention grabbed]

  Secondary (1-3s):
    Dashboard UI becomes clear
    Multiple data sources visible
    [Context established]

  Tertiary (3-5s):
    Text appears: "Real-time attribution"
    [Message reinforced]
```

## Show Don't Tell Checklist

For each scene, ask:

### Could This Be Shown?

```
❌ "Our platform is fast"
✅ [Show: Side-by-side speed comparison]

❌ "Users love our product"
✅ [Show: Real testimonial with face/name + company]

❌ "Save time on repetitive tasks"
✅ [Show: Time-lapse of task automated, clock showing time saved]

❌ "Works with all your tools"
✅ [Show: Logos flowing into central platform]

❌ "Simple three-step process"
✅ [Show: Three steps in action, numbered]
```

### Is the Visual Carrying Its Weight?

**Good** (Visual adds value):
```
- Demonstrates functionality
- Shows transformation
- Provides proof
- Creates emotion
- Explains complex concept
```

**Bad** (Visual is decorative only):
```
- Generic stock footage
- Static logo
- Meaningless abstract shapes
- Unrelated B-roll
```

## Integration with Copy

### Copy Should Label, Not Describe

**Good**:
```
Visual: [Demo of instant sync across devices]
Copy: "Instant sync"

Copy labels what viewer sees
```

**Bad**:
```
Visual: [Static product logo]
Copy: "Our product provides instant synchronization across
all devices including desktop, mobile, and tablet platforms"

Copy does all the work
```

### Copy Should Amplify, Not Repeat

**Good**:
```
Visual: [Before/after comparison showing 10x speed increase]
Copy: "10x faster processing"

Copy quantifies what visual shows
```

**Bad**:
```
Visual: [Before/after comparison showing 10x speed increase]
Copy: "You can see here that the processing is much faster
than before"

Copy states the obvious
```

## Visual Storytelling by Video Type

### Product Demo (Show-Heavy)

```
Ratio: 70% show / 30% tell

Focus:
  - Screen recordings
  - Live interactions
  - Before/after states
  - Real workflows
```

### Brand Story (Emotion-Heavy)

```
Ratio: 50% show / 50% tell

Focus:
  - People and faces
  - Real moments
  - Emotional expressions
  - Journey visualization
```

### Social Proof (Balance)

```
Ratio: 60% show / 40% tell

Focus:
  - Real customer photos
  - Actual results/data
  - Company logos
  - Success metrics visualized
```

## Platform Considerations

### Social Media (Silent Viewing)

**Critical**: 60%+ watch without sound

```
MUST show:
  - Text overlays (readable, large)
  - Visual demonstrations
  - Clear before/after
  - Obvious transformations

AVOID:
  - Requiring audio to understand
  - Subtle visual details
  - Complex explanations
```

### Website Hero (Looping)

```
MUST show:
  - Product in action
  - Continuous loop (seamless)
  - Immediate value proposition
  - No beginning/end dependency

AVOID:
  - Linear narrative
  - Time-based progression
  - One-time reveals
```

## Anti-Patterns

### 1. Lorem Ipsum Visuals

**Bad**:
```
Generic stock footage of:
- Person typing on laptop
- Coffee cup on desk
- Handshake
- City skyline

None of which relate to actual product
```

### 2. Static Screenshots

**Bad**:
```
10 seconds of motionless screenshot
While text explains what it does
```

**Better**:
```
3 seconds of animated interaction
Text labels the benefit
```

### 3. Text Dumping

**Bad**:
```
Paragraph of text on screen
Minimal or generic visual
Viewer must read everything
```

**Better**:
```
Single sentence on screen
Visual demonstrates the point
Text + visual reinforce each other
```

### 4. Explaining the Visual

**Bad**:
```
Visual: [Data flowing between apps]
Copy: "As you can see, data flows between your apps..."
```

**Better**:
```
Visual: [Data flowing between apps]
Copy: "Automatic sync across all platforms"
```

### 5. Disconnected Visual/Copy

**Bad**:
```
Visual: [Generic office space]
Copy: "Our AI analyzes your data in real-time"

Visual and copy unrelated
```

**Better**:
```
Visual: [AI model analyzing data, visualized]
Copy: "Real-time AI analysis"

Visual shows what copy says
```

## Checklist

For each scene:

**Show Don't Tell**:
- [ ] Primary message shown visually (not just stated)
- [ ] Visual demonstrates/proves (not just decorates)
- [ ] Transformation visible (before/after, movement, change)
- [ ] Product shown working (not static screenshot)
- [ ] Data/numbers visualized (not just text)

**Visual Effectiveness**:
- [ ] Visual is specific to our product (not generic)
- [ ] Visual tells story even without sound
- [ ] Visual hierarchy clear (what to look at first)
- [ ] Motion directs attention appropriately
- [ ] Visual and copy reinforce each other

**Avoids Anti-Patterns**:
- [ ] No stock footage unrelated to product
- [ ] No static screenshots for extended duration
- [ ] No paragraph text blocks
- [ ] No copy explaining what's obvious visually
- [ ] No disconnected visual/copy pairs

**Platform Appropriate**:
- [ ] Works without sound (if social)
- [ ] Readable on mobile (if needed)
- [ ] Loops seamlessly (if hero video)
- [ ] Detail level matches viewing context
